Librarian Agent — System Architecture

Autonomous LLM-powered knowledge management • Telegram → LLM → GitHub Wiki

Tech Stack

  • • Python 3.11 · FastAPI · Uvicorn
  • • Pydantic v2 (structured LLM output)
  • • PyGithub · Git Data API (atomic commits)
  • • httpx (async Telegram calls)
  • • PyMuPDF4LLM (PDF → Markdown)
  • • Docker Compose · ngrok static domain
  • • python-dotenv · dataclass settings

API Endpoints

  • • POST /webhook/telegram — Telegram ingest
  • • POST /query — REST question answering
  • • WS /ws/query — streaming real-time query
  • • POST /lint — wiki orphan/ghost audit
  • • GET /health — liveness probe
  • • GET /docs — Swagger UI
  • • Auto-webhook registration on startup

Data & Pipelines

  • • 8-step ingest: validate → route (LLM) → fetch → compile (LLM) → commit → log → notify
  • • Dual LLM: Gemini (cloud) or Ollama (local)
  • • Large docs (>40k chars) condensed first
  • • Atomic single-commit via Git Tree API
  • • Wiki schema: sources/ entities/ concepts/ analyses/
  • • High-confidence Q&A auto-filed to analyses/
